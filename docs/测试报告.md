# 2048 控制台游戏测试报告

## 1 测试环境

| 项目 | 内容 |
|---|---|
| 操作系统 | Windows |
| 编辑器 | VSCode |
| 编译器 | g++ / MinGW-w64 |
| 语言标准 | C++17 |
| 测试对象 | 多文件版 `2048.exe` 与单文件版 `2048_single.exe` |

## 2 编译测试

多文件版编译命令：

```bash
g++ -std=c++17 -Wall -Wextra -Iinclude src/*.cpp -o 2048.exe
```

单文件版编译命令：

```bash
g++ -std=c++17 -Wall -Wextra single_file/main.cpp -o 2048_single.exe
```

两个版本均可正常编译。

## 3 测试说明

由于 2048 游戏中的新数字位置和数值由随机数决定，测试时不能要求每次棋盘完全一致。对于随机相关功能，主要观察以下现象：

1. 初始棋盘是否只有两个非零数字。
2. 新数字是否只出现在空白格。
3. 有效移动后是否生成新数字。
4. 无效移动后是否不生成新数字。

## 5 测试结论

本项目通过了启动、编译、四方向移动、合并、分数、随机生成、非法输入、胜利、失败、重新开始和退出等核心功能测试，满足 2048 控制台游戏的基本要求。

## 6 可能的问题与改进

1. 随机数导致每次初始棋盘不同，后续可增加测试专用接口以便自动化验证。
2. 当前最高分只在程序运行期间有效，后续可保存到本地文件。
3. 控制台界面较朴素，后续可扩展 EasyX 图形界面。
